# Product Requirements Document (PRD)
## 소설 쓰기 웹 애플리케이션 - MVP

### 1. 프로젝트 개요

**제품명**: Novel Writer Web App

**목적**: 작가들이 소설을 효율적으로 작성, 편집, 관리할 수 있는 웹 기반 플랫폼 제공

**타겟 사용자**:
- 아마추어 작가 및 취미 작가
- 프로 작가 및 웹소설 작가
- 창작 동호회 및 글쓰기 커뮤니티 멤버

---

### 2. 핵심 목표

1. **직관적인 글쓰기 환경** 제공으로 창작 몰입도 향상
2. **체계적인 작품 관리** 기능으로 대형 프로젝트 관리 용이
3. **크로스 플랫폼** 지원으로 언제 어디서나 작업 가능
4. **빠른 MVP 출시**로 사용자 피드백 수집

---

### 4. 비기능 요구사항

#### 4.1 성능
- 페이지 로딩 시간 < 2초
- 에디터 입력 지연 < 100ms
- 10,000자 이상 문서에서도 부드러운 스크롤

#### 4.2 보안
- HTTPS 암호화 통신
- JWT 기반 인증
- XSS, CSRF 방어
- 정기적인 데이터 백업 (일 1회)

#### 4.3 확장성 (MVP)
- 사용자당 최대 50개 프로젝트
- 프로젝트당 최대 500개 챕터
- 챕터당 최대 100,000자

#### 4.4 호환성
- 모던 브라우저 지원 (Chrome, Firefox, Safari, Edge)
- 모바일 반응형 디자인

### 5. 사용자 스토리 (MVP)

1. **작가로서** 새 소설 프로젝트를 생성하고, 챕터별로 내용을 작성할 수 있어야 한다.
2. **작가로서** 작성 중인 내용이 자동으로 저장되어 데이터 손실을 방지할 수 있어야 한다.
3. **작가로서** 작성한 소설을 TXT/DOCX 형식으로 내보낼 수 있어야 한다.
4. **작가로서** 내 총 단어 수와 챕터별 진행 상황을 확인할 수 있어야 한다.
5. **작가로서** 모바일 기기에서도 작업을 이어서 할 수 있어야 한다.

---

### 6. 성공 지표 (KPI) - MVP

- **사용자 증가율**: 월간 5% 증가
- **DAU/MAU 비율**: 15% 이상
- **평균 세션 시간**: 15분 이상
- **프로젝트 생성률**: 사용자당 평균 1개 이상
- **데이터 내보내기 비율**: 월간 활성 사용자의 20% 이상

### 7. 출시 계획

#### Phase 1 (MVP - 2-3개월) ⭐ 현재 범위

**핵심 기능**:
- 기본 리치 텍스트 에디터 (Tiptap)
- 프로젝트/챕터 CRUD
- 사용자 인증 (이메일 + Google/Kakao)
- 자동 저장 (5초)
- 단어 수 카운팅
- TXT/DOCX 내보내기
- 간단한 통계 (총 단어 수, 챕터별)
- 반응형 디자인
- 다크모드

**목표**:
- 빠른 출시로 사용자 피드백 수집
- 핵심 가치 검증
- 초기 사용자 확보 (100-500명)

#### Phase 2 (4-6개월)

**추가 기능**:
- 캐릭터/세계관 관리
- 폴더 구조
- PDF 내보내기
- 통계 대시보드 (그래프)
- 읽기 전용 공유 + 댓글
- 드래그 앤 드롭 챕터 순서 변경
- 집중 모드

#### Phase 3 (6-12개월)

**고급 기능**:
- ePub 내보내기
- 오프라인 지원
- (선택) 실시간 협업
- (선택) AI 어시스턴트

---

### 8. 제약사항 및 리스크

**제약사항**:
- 초기 개발 리소스 제한 (1-2명)
- 개발 기간 제한 (2-3개월)
- 예산 제한

**리스크**:
- 경쟁 서비스 (Scrivener, Google Docs, Notion 등)
- 사용자 데이터 보안 이슈
- 에디터 성능 문제 (대용량 텍스트)

**완화 전략**:
- MVP에 집중하여 빠른 출시
- 자동 백업 강화
- 가상 스크롤로 성능 최적화
- 모니터링 시스템 구축

---

### 9. MVP 제외 기능 (중요!)

다음 기능들은 **구현 난이도가 높아 MVP에서 제외**합니다:

**제외 목록**:
1. ❌ **실시간 협업** (공동 편집) - Phase 3로 연기
2. ❌ **ePub 내보내기** - Phase 3로 연기
3. ❌ **AI 기능** (문법 검사, TTS, 음성 입력) - Phase 3로 연기
4. ❌ **오프라인 지원** - Phase 3로 연기
5. ❌ **버전 히스토리** - Phase 2로 간소화 연기
6. ❌ **플롯 타임라인** - Phase 3로 연기
7. ❌ **관계도 시각화** - Phase 3로 연기

**제외 이유**: 개발 기간 단축 및 핵심 가치 집중
